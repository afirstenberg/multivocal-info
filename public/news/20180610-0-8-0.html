Multivocal 0.8.0 now available

Details below, but the big focus for this update was about identity and authentication.

Multivocal is a library to help you build great agents for the Google Assistant (and other platforms in the future). We take a different approach - letting you focus on what you want to do and say, rather than the boilerplate. Learn more at https://multivocal.info/

Updates in 0.8.0:
- A UserID will be available in the environment path User/Id. This will be true even when Actions on Google discontinue their support for a native UserID.
- If you're doing Account Linking with your own OAuth2 server (or a server such as auth0), the access token will be available in User/AccessToken and User/IsAuthenticated will be set to true.
- If you're using Google Sign In for the Assistant, the raw ID token will be available in User/IdentityToken. If this token is validated, the profile will be in User/Profile and User/IsAuthenticated will be set to true.

We have some examples in the pipleline, and they'll be available within the next few days.
