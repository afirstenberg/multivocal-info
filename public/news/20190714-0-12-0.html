Multivocal 0.12.0 now available

The main update in this release is support for postprocessing - code that will run after the reply is sent. This is particularly useful for operations, such as database saves or logging, that don't need to take time before the reply.

Also notable, tho largely internal:

  • Record timing for various stages of processing. (And report these as part of postprocessing by default.)
  • Performance improvements for configuration handling.
